# EventSnap — инструкции для агентов

## Документация

После любых изменений в коде обновляй соответствующие файлы в `docs/`:

| Что изменил | Что обновить |
|---|---|
| Модели, схема БД, миграции | `docs/technical.md` — раздел "Модели БД" |
| Новые зависимости (requirements.txt) | `docs/technical.md` — раздел "Стек" |
| Новые модули, изменение структуры `src/` | `docs/technical.md` — раздел "Архитектура" |
| Новая функциональность | `docs/mvp.md` — добавить в список требований |
| Архитектурные решения, паттерны | `docs/technical.md` — соответствующий раздел |

Правило: если изменение влияет на то, как устроен проект — документация обновляется в том же ответе, не откладывая.
